# Methodology Notes

This folder documents the methodological layer behind the landing page narrative.

## What this means in this repository

For this project, "methodology" does **not** mean a full open-source simulation stack. It means the public explanation of how Dasein frames its work:

- signal reading
- actor mapping
- scenario logic
- decision support under uncertainty

## What is currently grounded here

- a research note on signal-count framing:
  [`docs/research-signal-count-methodology.md`](../docs/research-signal-count-methodology.md)
- the public site copy and information architecture in `src/`
- public offer framing in the README and status docs

## Limits

- The repo does not yet publish a public case study that traces method to result.
- The repo does not expose the internal simulation runtime.
- The repo should not be read as evidence that Dasein has solved forecasting, only that it has a clear public position on how to approach uncertainty.
- Some claims are strategic framing, not validated benchmarks.

## What would strengthen this section

- one public case narrative
- one annotated methodology walkthrough
- one concrete artifact showing how signals become a recommendation or scenario brief
