# Roadmap

This roadmap is intentionally small. The goal is not feature inflation. The goal is to make the repository clearer, more defensible, and more useful as a public artifact.

## Now

- rewrite the README around the real project identity
- add status, methodology, and screenshot documentation
- remove obvious presentation clutter from the root

## Next

- add a clean screenshot set for desktop and mobile
- publish one public case or capability note
- add an Open Graph preview image for the repository and site
- isolate or remove tracked internal assistant/reference files from the public root

## Later

- add a short public methodology walkthrough
- publish at least one concrete artifact from a real engagement
- decide whether the repo should remain a pure site repo or expand into a broader public venture repository

## Explicit non-goals

- pretending this is already a mature product platform
- stuffing the repo with speculative features
- turning the site into a generic startup template
