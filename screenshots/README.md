# Screenshots

This directory is for the images that make the repository immediately legible on GitHub.

## Recommended captures

Current files in this directory should include:

- `home-desktop.png`
  Desktop homepage above the fold. Capture the hero, navigation, and first proof of visual quality.
- `services-desktop.png`
  Desktop view of the services section. This shows the offer clearly.
- `home-mobile.png`
  Mobile homepage view. This proves the site is considered on smaller screens.

## Capture guidance

- Use production or preview mode, not an in-editor screenshot.
- Hide browser chrome if possible.
- Prefer light compression over blurry images.
- Capture the actual site, not design mockups.

## Ideal dimensions

- Desktop: `1600px` to `1800px` wide
- Mobile: `430px` to `460px` wide

## Why these matter

Without screenshots, the repo asks visitors to imagine the result. A pinned repo should reduce that gap.
